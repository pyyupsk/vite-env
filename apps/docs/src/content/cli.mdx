The `@vite-env/cli` package provides three subcommands for use outside the Vite build pipeline.

```bash
npm install -D @vite-env/cli
```

All subcommands resolve `env.ts` from the current working directory using `jiti` (TypeScript support at runtime).

## check

Validates the current environment against your schema and exits non-zero on failure. Useful in CI before deploying.

```bash
vite-env check
vite-env check --config ./env.ts --mode production
```

| Flag       | Default       | Description                                              |
| ---------- | ------------- | -------------------------------------------------------- |
| `--config` | `env.ts`      | Path to env definition file                              |
| `--mode`   | `development` | Vite mode (affects which `.env.[mode]` files are loaded) |

Output on success:

```
✔ 5 environment variables valid
```

Output on failure:

```
✖ Environment validation failed:
  DATABASE_URL: Invalid url
  SECRET_KEY: String must contain at least 32 character(s)
```

## generate

Generates a `.env.example` file from your schema. Useful for onboarding — shows every required key with its type hint and default value.

```bash
vite-env generate
vite-env generate --config ./env.ts --output .env.example
```

| Flag       | Default        | Description                 |
| ---------- | -------------- | --------------------------- |
| `--config` | `env.ts`       | Path to env definition file |
| `--output` | `.env.example` | Output path                 |

Example output:

```bash
# Auto-generated by @vite-env/cli — do not edit manually
# Run: npx vite-env generate

# ── Client variables (VITE_ prefix — available everywhere)
# VITE_API_URL: string — required
VITE_API_URL=

# ── Server-only variables (never sent to browser)
# DATABASE_URL: string — required
DATABASE_URL=
# PORT: number
PORT=3000
```

## types

Regenerates `vite-env.d.ts` from your schema without running a full Vite build. Run this after changing your schema to update editor types immediately.

```bash
vite-env types
vite-env types --config ./env.ts
```

| Flag       | Default  | Description                 |
| ---------- | -------- | --------------------------- |
| `--config` | `env.ts` | Path to env definition file |

Writes `vite-env.d.ts` to the current working directory. The plugin also writes this file automatically on every `buildStart`, so manual invocation is only needed outside a Vite workflow.
